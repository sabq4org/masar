# مسار — منصة إدارة العمل والمهام بنموذج أسانا

منتج مستقل لإدارة العمل والمشاريع، مبني على نموذج أسانا: مهمة تكتمل بعلامة ✓، أقسام حرة تتنقل المهام بينها بالسحب، لوحة تفاصيل جانبية، وصفحة «مهامي» بأقسام شخصية.

## التشغيل

```bash
npm install            # إلزامي بعد كل سحب — غياب حزم مثل multer يُسقط الخادم بالكامل
cp .env.example .env   # DATABASE_URL و SESSION_SECRET و ADMIN_PASSWORD
npm run db:push        # دفع المخطط إلى القاعدة
npm run seed           # الفرق + حساب المدير + أقسام «مهامي»
npm run seed:demo      # (اختياري) مشروعان تجريبيان و١٦ مهمة و٣ مستخدمين
npm run dev            # الخادم على 5180 والواجهة على 5181
```

**مهم:** الواجهة على `5181` والـ API على `5180`. إن رأيت «تُضاف ولا تظهر» فتأكد أن `npm run dev` شغّال الطرفين (ليس Vite وحده).

**السرعة محليًا:** قاعدة Neon البعيدة تضيف ~١–١.٥ ثانية لكل طلب. للتطوير السريع استخدم Postgres محلي في `DATABASE_URL` (مثل `postgresql://sabq:…@localhost:5432/masar`).

الدخول الأولي: البريد وكلمة المرور من `ADMIN_EMAIL` / `ADMIN_PASSWORD` في `.env` — **غيّر كلمة المرور بعد أول دخول**.

## الترقية من النسخة السابقة (نموذج الـ١٣ حالة)

إن كانت قاعدتك تحمل بيانات من النسخة القديمة:

```bash
npm run migrate:asana   # ينقل البيانات: الحالات → مكتملة/غير مكتملة، subtasks → مهام كاملة…
npm run sync:asana      # يجعل status_id اختياريًا + ينشئ الجداول الناقصة (آمن، بلا حذف)
npm run db:push         # (اختياري لاحقًا) وافق على إسقاط الجداول القديمة statuses/subtasks…
npm run seed
```

## البنية

- `shared/schema.ts` — Drizzle: users, departments, user_task_sections, projects, project_sections, project_members, project_stars, project_status_updates, tasks (بمهام فرعية عبر parent_task_id), task_comments, task_watchers, task_likes, comment_likes, task_activity, task_dependencies, attachments, project_templates, notifications, session
- `server/` — Express: جلسات + bcrypt، صلاحيات أدوار ثابتة (`permissions.ts`)، مسارات مقسمة في `routes/`، رفع مرفقات multer إلى `uploads/`، SSE، تذكيرات، AI
- `client/` — React + Vite + Tailwind، عربي RTL، هوية «الخطّ الساري»
  - `components/TaskList.tsx` و`TaskBoard.tsx` — عرضا القائمة واللوحة العامّان (يخدمان مهامي والمشاريع معًا)
  - `components/TaskPane.tsx` — لوحة تفاصيل المهمة (تفتح من أي صفحة عبر `lib/taskPane.tsx`)

## قرارات معمارية

- **نموذج أسانا للمهمة**: `is_completed` مصدر الحقيقة؛ لا آلة حالات. الأقسام أعمدة حرة.
- **المهام الفرعية مهام كاملة** عبر `parent_task_id` — لها مسؤول واستحقاق وتعليقات.
- **مهمة الاعتماد** نوع مهمة (`task_type = approval`): اعتماد/رفض يكملها، طلب التعديل لا يغلقها.
- **أقسام «مهامي» شخصية** لكل مستخدم؛ تغيير المسؤول يعيد المهمة إلى «المسندة حديثًا» عند المسؤول الجديد.
- **مسؤول واحد لكل مهمة** + متعاونون (`task_watchers`).
- **مشروع واحد لكل مهمة** بأعمدة مباشرة؛ الإسكان المتعدد لاحقًا عبر `placements`.
- **التواريخ ميلادية** بواجهة عربية (`ar-SA-u-ca-gregory`).

## المراحل القادمة

حقول مخصصة، مهام متكررة، إسكان المهمة في عدة مشاريع، قواعد أتمتة (Rules)، نماذج استقبال الطلبات (Forms)، محافظ المشاريع (Portfolios)، أهداف (Goals)، تعدد المساحات، PWA أعمق.

## تطبيق iOS (`ios/`)

تطبيق SwiftUI أصلي (iOS 17+) بهوية «الخطّ الساري» كاملة — يستهلك نفس REST API. الشاشات: دخول، الرئيسية، مهامي (قائمة بسحب-للإكمال + لوحة)، تفاصيل المهمة (اعتماد/إعجاب/فرعيات/تعليقات/سجل)، الوارد (أنشطة/أرشيف)، المشاريع وصفحة المشروع (نظرة عامة بتحديثات الحالة + قائمة + لوحة)، البحث، والإنشاء السريع. الخطوط مضمّنة (Alexandria وIBM Plex Sans Arabic وSpace Grotesk)، والمشروع بمجموعة ملفات متزامنة — أي ملف Swift جديد داخل `ios/Masar/` يدخل البناء تلقائيًا بلا تعديل pbxproj. الفتح: `open ios/Masar.xcodeproj`، وعنوان الخادم في `Core/API.swift`.
