# مسار — نظام إدارة العمل والمهام لغرفة الأخبار

منتج مستقل لإدارة دورة العمل التحريري — من الفكرة إلى النشر. قاعدة بيانات Neon خاصة به، بلا اعتماد على أنظمة نشر خارجية.

## التشغيل

```bash
npm install
cp .env.example .env   # DATABASE_URL و SESSION_SECRET و ADMIN_PASSWORD
npm run db:push        # دفع المخطط إلى القاعدة
npm run seed           # الحالات + الانتقالات + الأقسام + حساب المدير
npm run dev            # الخادم على 5180 والواجهة على 5181
npm run test:rules     # اختبارات محرك انتقال الحالات
```

الدخول الأولي: البريد وكلمة المرور من `ADMIN_EMAIL` / `ADMIN_PASSWORD` في `.env` — **غيّر كلمة المرور بعد أول دخول**.

## البنية

- `shared/schema.ts` — Drizzle (users, departments, statuses, transitions, projects, sections, tasks, subtasks, comments, watchers, activity, dependencies, approvals, templates, notifications + sessions)
- `shared/statusRules.ts` — منطق انتقال الحالات النقي (مختبر)
- `server/` — Express: جلسات + bcrypt، صلاحيات أدوار ثابتة، خدمات المهام، تذكيرات، SSE، AI
- `client/` — React + Vite + Tailwind، عربي RTL، هوية «الخطّ الساري»

## قرارات معمارية

- **الحالة مصدر حقيقة واحد**: 13 حالة بفئات (planning/active/blocked/review/done/closed). الانتقال إلى مؤجلة/ملغاة من أي حالة؛ الخروج منهما إلى أي غير مغلقة؛ الباقي بمصفوفة `status_transitions`.
- **مسؤول واحد لكل مهمة** + مشاركون (`task_watchers`).
- **مشروع واحد لكل مهمة** بأعمدة مباشرة؛ الإسكان المتعدد لاحقًا عبر `placements`.
- **الأدوار خريطة كود** (`server/permissions.ts`).
- **طلب التعديل في المراجعة لا يغلق المهمة**.
- **منتج مستقل** — روابط مرجعية حرة (`linkUrl`) بلا تكامل أنظمة خارجية.

## اختصارات لوحة المفاتيح

| مفتاح | الفعل |
|-------|--------|
| `N` | التركيز على إضافة مهمة سريعة (مهامي) |
| `/` | التركيز على بحث المشروع |
| `Esc` | إغلاق لوحة تفاصيل المهمة |
